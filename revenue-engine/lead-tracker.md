# Lead Tracker

> Use this as a Kanban-style pipeline template. Copy to Notion, Airtable, GitHub Projects, or any CRM.
> Update **Next Action** and **Stage** after every interaction.

---

## Pipeline Stages

| Stage | Definition |
|-------|-----------|
| **Identified** | Company/contact found; no outreach yet |
| **Contacted** | First message sent; awaiting response |
| **Demo** | Discovery call scheduled or completed |
| **Proposal** | Scope/pricing sent; in negotiation |
| **Closed — Won** | Deal signed |
| **Closed — Lost** | No deal; reason logged |
| **Nurture** | Not ready now; follow up in 30–90 days |

## Field Definitions

| Field | Description |
|-------|-------------|
| **Company Type** | Describe category + stage + rough ARR — don't need real company name until deal is active |
| **Contact Role** | Job title or function of the champion / decision-maker |
| **Interest Signal** | What triggered identification — GitHub activity, LinkedIn, referral, inbound |
| **Deal Size Est.** | License tier × annual price, or consulting hours × rate estimate |
| **Next Action** | Specific, dated action — never leave this blank |
| **Owner** | Team member or agent responsible for next action |

## Update Cadence

- **Weekly:** Review all Contacted and Demo stage entries; send follow-ups per sequence
- **Bi-weekly:** Sweep Identified for new companies; check GitHub fork activity
- **Monthly:** Review Nurture list; advance or close-lost stale entries
- **Quarterly:** Reconcile closed-won against revenue targets in `/docs/revenue-engine/README.md`
